## Endpoint komend

Na start wystarczy ogГіlny endpoint `POST /commands`, ktГіry przyjmuje envelope i deleguje do command busa. W praktycznych implementacjach Python CQRS czД™sto zaczyna siД™ wЕ‚aЕ›nie od jawnego command handlera i event store.[^1][^3]

## Shell agent

Worker shellowy powinien subskrybowaД‡ temat NATS, wykonywaД‡ zadanie i odsyЕ‚aД‡ wynik jako kolejnД… komendД™ do orchestratora. NATS w Pythonie dobrze nadaje siД™ do prostego modelu request/reply albo pub/sub dla taskГіw rozdzielanych miД™dzy workery.[^4][^10][^5]

Minimalny `agents/shell-agent/app/main.py`:

```python
import asyncio
import json
import os
import subprocess
from nats.aio.client import Client as NATS

NATS_URL = os.getenv("NATS_URL", "nats://nats:4222")
AGENT_ID = os.getenv("AGENT_ID", "shell-agent-a")

async def main():
    nc = NATS()
    await nc.connect(NATS_URL)

    async def handle(msg):
        data = json.loads(msg.data.decode())
        task_id = data["task_id"]
        command = data["command"]

        proc = subprocess.run(command, shell=True, capture_output=True, text=True, timeout=120)
        event = {
            "agent_id": AGENT_ID,
            "task_id": task_id,
            "exit_code": proc.returncode,
            "stdout": proc.stdout[-4000:],
            "stderr": proc.stderr[-4000:]
        }
        subject = "task.shell.completed" if proc.returncode == 0 else "task.shell.failed"
        await nc.publish(subject, json.dumps(event).encode())

    await nc.subscribe("task.assigned.shell", cb=handle)
    while True:
        await asyncio.sleep(1)

if __name__ == "__main__":
    asyncio.run(main())
```

Na MVP to wystarczy, Ејeby uruchamiaД‡ shell taski i zwracaД‡ outcome.[^10][^5][^4]
